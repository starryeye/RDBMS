## VIEW
- 뷰에 대해 알아본다.
  - 뷰의 필요성
    - 어떤 복잡한 조회 쿼리를 저장해놓고 간단하게 사용하고 싶음..
    - 원본 테이블의 접근 권한은 줄 수 없지만, 원본 테이블에서 특정 데이터를 가공한 데이터들의 조회 권한은 줘야할 때..
  - 뷰
    - 실제 데이터를 가지고 있지 않은 가상의 테이블이다.
    - 복잡한 SELECT 문을 가지고 있다.
    - SELECT * FROM {뷰 이름}; 을 수행하면, 뷰가 가진 복잡한 SELECT 문을 실행한 결과를 반환해준다.
      - 마치 어떤 테이블을 조회하는 것처럼 느껴짐
      - 원본 테이블위에 추상화 계층이 생긴느낌
  - 주의 사항
    - 뷰에 뷰를 사용하는 중첩 구조는 성능 저하의 주된 원인이므로 피해야한다.
    - 뷰를 (가상의) 테이블 처럼 다룰 수 있지만..
      - 뷰가 가진 SELECT 문에 JOIN, 집계함수, GROUP BY, DISTINCT 등이 사용되었다면..
        - VIEW 에 INSERT, UPDATE, DELETE 를 사용할 수 없다.
        - VIEW 에 CUD 를 사용하면 원본 데이터가 변경되지만, 애초에 VIEW 에 CUD 를 사용하는 것은 절대 삼가해야함
