## JOIN
- 조인이 필요한 이유
  - 여러 정보가 정규화를 거쳐 여러 테이블에 나누어져 있는데 이를 한번에 조회하고 싶음
  - 즉, 데이터 정규화(분리)를 통해 일관성과 효율성은 얻으면서 조인으로 통합 결과를 얻기 위해서임
- 테이블 정규화가 필요한 이유
  - 데이터의 중복을 막고, 일관성을 지키기 위해서이다.
  - 정규화를 하지 않으면 아래의 문제가 나타난다.
    - 데이터 중복
      - 주문 데이터와 유저 데이터가 구분없이 한 테이블에 저장되면, 하나의 주문에 유저 데이터가 중복되어 저장됨
    - 갱신 이상
      - 주문 데이터와 유저 데이터가 구분없이 한 테이블에 저장되면, 유저 정보를 수정한 경우 모든 주문 데이터에 갱신이 이루어 져야함
    - 삽입 이상
      - 주문 데이터와 상품 데이터가 구분없이 한 테이블에 저장되면, 주문이 이루어지지 않은 상품은 DB 에 저장할 수 없다
    - 삭제 이상
      - 주문 데이터와 유저 데이터가 구분없이 한 테이블에 저장되면, 주문 데이터를 삭제하면 유저 데이터도 함께 삭제된다
- 조인 방향에 따라 데이터가 늘어나기도 한다.
  - 기준으로 잡을 테이블(FROM) 선정이 중요하다. 
  - ToOne
    - FROM 테이블에서 JOIN 테이블 방향으로 FK -> PK 라면, row 수가 늘어나지 않음
  - ToMany
    - FROM 테이블에서 JOIN 테이블 방향으로 PK -> FK 라면, row 수가 늘어난다.
