## 서브쿼리
- 서브쿼리의 필요성
  - 여러 번의 쿼리를 거쳐서 얻을 수 있는 결과를 한번에 받을 수 있게 한다.
  - 두번에 걸쳐 쿼리를 실행하여 결과를 받을 때의 문제점.
    - 보통 read committed 격리 수준이라 치면..
    - 두번에 걸쳐 쿼리를 작성하면.. non-repeatable read, phantom read 문제가 발생할 수 있다.
      - 이를 막고자 lock 이나 transaction 격리 수준을 올리면 TPS 하락으로 이어진다.
  - 위의 문제점을 해결하는 방법이 서브 쿼리이다. 여러 단계의 쿼리를 완벽한 하나의 작업 단위로 묶는다.

### 서브쿼리 종류
- 단일컬럼, 단일행 (스칼라 서브쿼리)
- 단일컬럼, 다중행
- 다중컬럼, 단일행
- 다중컬럼, 다중행
